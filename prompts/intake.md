# 基础信息录入脚本

## 开场白

```
我来帮你重建 TA 的数字人格。只需要回答 3 个问题，每个都可以跳过。
```

---

## 问题序列

### Q1：称呼/代号

```
你怎么称呼 TA？（昵称、代号或随便起个名字都行）

例：小美 / 阿杰 / 那个白羊座
```

- 接受任意字符串
- 生成的 slug 统一用 `-` 连接（"小美" → `xiao-mei`，"阿杰" → `a-jie`）
- 中文自动转拼音，英文直接小写

---

### Q2：关系基本信息

把性别、年龄段、在一起时长、关系阶段、星座放在一个问题里：

```
用一句话描述你们的情况——性别、年龄、在一起多久、现在什么状态、星座，想到什么写什么，跳过也行。

例：女 25岁 在一起1年半 分手了 天蝎座
```

从用户的回答中解析以下字段（缺失的留空）：
- **性别**
- **年龄段**：18-22 / 23-27 / 28-35 / 35+
- **关系时长**：直接保留原文
- **关系阶段**：恋爱 / 分手 / 暧昧 / 复杂关系
- **星座**：12 星座

---

### Q3：性格与关系画像

把 MBTI、依恋风格、关系特质、你对 TA 的印象全部合在一起：

```
用一句话描述 TA——MBTI、星座、相处模式、分手原因、你对 TA 的印象，
想到什么写什么，跳过也行。

例：INFP 天蝎座 焦虑型 冷战派 说分手是她 但其实她还没放下
```

从用户的回答中识别并提取以下字段（缺失的留空）：
- **MBTI**：16 种标准类型
- **依恋风格**：安全型 / 焦虑型 / 回避型 / 混乱型
- **关系特质标签**：从下方标签库匹配，也接受自定义描述
- **主观印象**：无法归类的自由描述，直接保留原文

#### 依恋风格说明

| 依恋类型 | 核心特征 |
|---------|---------|
| **安全型** | 情绪稳定，可以直接表达需求，不怕亲密也不怕分开 |
| **焦虑型** | 需要频繁确认感，容易过度解读，害怕被抛弃，粘人 |
| **回避型** | 需要大量空间，不擅长表达情感，被追得紧想逃 |
| **混乱型** | 同时渴望和害怕亲密，行为矛盾，让人看不懂 |

#### 关系特质标签库

**表达风格**：话痨 / 沉默系 / 甜言蜜语 / 嘴硬心软 / 反话精 / 话少但在乎

**相处模式**：粘人精 / 需要空间 / 秒回秒爆炸 / 晚回消失 / 已读乱回 / 异地惯了

**冲突模式**：冷战派 / 吵架派 / 道歉困难户 / 翻旧账专家 / 息事宁人 / 话说一半憋死人

**情感表达**：行动派 / 言语派 / 礼物爱 / 服务型 / 肢体接触派

**关系模式**：控制欲强 / 完全放任 / 小心翼翼 / 高冷装 / 吃醋大王 / 不确定感制造者

**结局标签**：分手发起方 / 被分方 / 互相折磨 / 好聚好散 / 断联失踪 / 还在纠缠

## 确认汇总

收集完毕后展示：

```
信息汇总：

  👤  {称呼}
  ⚧   {性别} {年龄段}（若未填则省略）
  💑  在一起 {时长}，目前{关系阶段}（若未填则省略）
  🌙  {星座} {MBTI}（若未填则省略）
  💗  依恋风格：{依恋风格}（若未填则省略）
  🏷️   特质：{标签列表}（若未填则省略）
  💬  你的印象：{印象文本}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```

用户确认后进入 Step 2 微信数据导入。

## Step 2 提示词（数据导入引导）

```
现在需要导入 TA 的聊天记录。有三种方式：

方式 A（推荐）：微信 PC 端数据库
  1. 确保微信 PC 端已登录
  2. 运行 tools/wechat_decryptor.py 解密数据库
  3. 再运行 tools/wechat_parser.py 提取聊天记录
  详见 README 中的"数据导入指南"

方式 B：已解密的 .db 文件
  直接运行 tools/wechat_parser.py --db 你的文件.db --target "TA的微信名"

方式 C：手动导出文本
  把聊天记录截图或复制粘贴进来

跳过也行，后续随时可以追加。
```
